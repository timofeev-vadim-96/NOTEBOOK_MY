> Что делает команда git cherry-pick?

Для копирования выбранного коммита из одной ветки в конец другой в виде нового коммита. При этом, вторая ветка остается прежней

> Почему не следует использовать git push с флагом --force в реальной разработке?

Потому что, в таком случае, локальное состояние истории коммитов перетирает состояние в удаленном репозитории не взирая на возможные конфликты

> Что делает команда git remote -v?

Отображает все связи с удаленными репозиториями

> Как создать новую ветку и перейти в неё?

git branch, или git checkout -b name remoteName

> Как посмотреть историю изменений и сравнить версии файлов?  

Посмотреть историю изменений можно командой git log, а сравнивать версии — командой git diff

> Как внести доп. изменения в последний коммит без создания нового?

git commit --amend

> Как откатить безвозвратно изменения до какого-то определенного коммита, а как просто откатить конкретный коммит с сохранением в истории коммитов? 

git reset 12345 - безвозвратно, gire revert 12345 - откатить конкретный коммит

> Как сделать так, чтобы изменения из другой ветки оказались ниже изменений, сделанных в нашей текущей ветке?

git rebase

> Что предпринять, чтобы при предотвратить конфликт между ветками при создании пулл-реквеста?

Перед созданием пулл-реквеста, сделать git pull из ветки, в которую собираетесь делать пулл-реквест, разрешить конфликты, выполнить git push и создать пулл-реквест
